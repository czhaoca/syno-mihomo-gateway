# v1.5.0 — group-model streamline

Pick your exit country from ONE selector, human names on every card you
touch, and the Priority Nodes filter machinery removed. **This release
requires a one-time hand-edit of your `.env` — read the checklist below
before deploying.**

## What changed

Group model, before → after (mihomo group names ARE the dashboard labels;
dashboard order = definition order):

| v1.4.0            | v1.5.0                          | Role |
| ----------------- | ------------------------------- | ---- |
| PROXY             | **Proxy Mode**                  | The mode selector and rule target (GEOSITE / MATCH): Country Pick (default), DIRECT, REJECT, plus every raw provider node |
| STREAMING         | **Streaming Sites**             | Per-service selector for unlock-sensitive sites (Netflix + Spotify/Tidal/Deezer/SoundCloud rules): Proxy Mode (default), the country groups, DIRECT, raw nodes |
| —                 | **Country Pick** (NEW)          | Choose the exit COUNTRY; its members are exactly the generated `<Country> Auto` groups; defaults to your FIRST `COUNTRY_GROUPS` entry |
| `<Country> Auto`  | `<Country> Auto` (unchanged)    | Same url-test mechanics: filter regex, empty-fallback REJECT, tolerance 50, provider health check |
| All Nodes         | All Nodes, now `hidden: true`   | Kept SOLELY as the DNS detour anchor (the `#All Nodes` fragments in `DNS_FOREIGN_NAMESERVER`); no longer shown in MetaCubeXD |
| Priority Nodes    | **removed**                     | Along with the `PRIORITY_INCLUDE_FILTER` / `PRIORITY_EXCLUDE_FILTER` knobs |

Why: a stable exit country should be a first-class CHOICE, not a regex you
tune. Country Pick rides the ONE node the selected country's url-test
currently holds, so latency switching never hops your egress country on its
own — the hopping that trips site bot-protection. Within-country re-picks
still happen (tolerance 50, provider health-check cadence); if a site
objects to same-country IP hops too, pin one raw provider node in Proxy
Mode — that is the complete fix. The naming legend used across the
dashboard: `<X> Auto` = auto-pick url-test, `<X> Mode` = mode selector,
`<X> Sites` = site-rule group, `<X> Pick` = manual choice.

Two smaller changes ride along:

- **Doctor**: `proxy_groups` now judges the country Country Pick is
  actually riding — that group empty is `default-empty` (bad: tune the
  `COUNTRY_GROUPS` regexes, or pick another country in Country Pick as a
  stopgap); some OTHER country group empty is `country-empty` (warn);
  everything empty is `provider-empty` (cold/dead provider — seed it). The
  Priority-related states are gone.
- **Renderer guard**: every `#group` fragment in any `DNS_*` list must name
  a group the render actually produces (or DIRECT) — a dangling fragment
  refuses the render instead of dead-ending resolvers at runtime.

## REQUIRED: one-time .env hand-edit (before deploying this release)

> **Status: applied.** This edit was performed on the only known deployment
> (the gateway NAS) during this release's on-device validation (2026-07-17
> CST), via a staged throwaway script deleted afterward — no migration
> machinery ships in the repo. The checklist below stays for anyone
> upgrading another pre-v1.5.0 install; fresh installs start from
> `.env.example` and never need it.

There is no migration machinery — edit `../syno-mihomo-gateway-data/.env`
by hand:

1. **Delete the Priority knobs** — remove the `PRIORITY_INCLUDE_FILTER=`
   and `PRIORITY_EXCLUDE_FILTER=` lines (and any `AUTO_EXCLUDE_FILTER=`
   leftover from a pre-v1.4.0 install). A leftover line **refuses the
   render** with a removal-instruction error.
2. **Confirm `COUNTRY_GROUPS` is set** — it is REQUIRED now (Country
   Pick's members ARE the generated groups); empty or unset refuses the
   render, pointing at the `.env.example` default. That shipped default now
   puts **Japan Auto first** — the first entry is the out-of-box default
   exit country, preserving v1.4.0's Japan default. An existing `.env`
   keeps its own entry order, so YOUR first entry is YOUR default country —
   reorder if you want a different one.
3. **`DNS_FOREIGN_NAMESERVER` needs NO change** — the `#All Nodes`
   fragments keep working as-is: the group survives, just hidden from the
   dashboard.

Fail-loud, LAN-safe: a stale knob, a missing `COUNTRY_GROUPS`, or a
dangling `#group` fragment refuses the render, and the entrypoint gate
keeps the previous config running until you fix the `.env` — the LAN never
goes down, but your edit is silently NOT live. The doctor's
`config_rejected` check surfaces exactly that state.

## Expect once after upgrading

- **Dashboard pins reset once**: mihomo's `cache.db` keys selections by
  group NAME, so every renamed or removed group falls back to its first
  member. If you had a streaming-unlock node pinned on STREAMING, re-pin it
  on **Streaming Sites** in MetaCubeXD after the first start (and re-make
  any other custom selections).

## Upgrading from before v1.4.0

Apply `docs/release-notes/v1.4.0.md` first (it ships in the bundle), then
the checklist above.
