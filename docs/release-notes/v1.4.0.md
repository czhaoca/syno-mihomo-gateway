# v1.4.0 — proxy groups v2

Human dashboard labels, a region-stable default route, audio streaming on the
STREAMING selector, and a no-back-compat purge. **This release requires a
one-time hand-edit of your `.env` — read the checklist below before
deploying.**

## What changed

- **Groups renamed** (mihomo group names ARE the dashboard labels):
  `auto` → **All Nodes**, `auto-x` → **Priority Nodes**, and the shipped
  country groups are now English: **US Auto, Singapore Auto, Japan Auto,
  Taiwan Auto, Philippines Auto**. Group names may now contain interior
  spaces.
- **Region-stable default route**: the new `PRIORITY_INCLUDE_FILTER`
  (ships scoped to Japan: `日本|JP\d|^JP`) keeps only matching nodes in
  Priority Nodes *before* `PRIORITY_EXCLUDE_FILTER` (ships the HK
  exclusion) subtracts — latency switching no longer hops your egress
  country, so login-sensitive sites stop flagging the account. Tune the
  include pattern to YOUR airport's naming (city-only pools like 东京/大阪
  need those words added); a zero-match pool REJECTs (fail closed, the
  doctor reports `proxy_groups: default-empty`).
- **STREAMING selector** now offers PROXY (default), Priority Nodes,
  All Nodes, every country group, and DIRECT — one-click Netflix-region
  pinning. **Audio streaming rides STREAMING too**: Spotify, Tidal, Deezer
  and SoundCloud rules land next to the Netflix rule. (Apple Music is
  deliberately excluded — CN storefront risk; YouTube Music follows
  `geosite:youtube` → PROXY.)
- **Purged** (single-NAS deployment, no upgrade compat maintained): the
  legacy flat-install importer (`migrate_legacy.sh`) and its machinery, the
  provider-cache md5 adoption, the DNS-privacy upgrade wizard, and the
  entire legacy DNS render profile — **split-horizon v2 is the only DNS
  profile**; `DNS_CN_NAMESERVER` + `DNS_FOREIGN_NAMESERVER` are REQUIRED
  and `DNS_FALLBACK` no longer exists.

## REQUIRED: one-time .env hand-edit (before deploying this release)

There is no migration machinery — edit `../syno-mihomo-gateway-data/.env`
by hand:

1. **Rename the exclude knob** — change the `AUTO_EXCLUDE_FILTER=` key to
   `PRIORITY_EXCLUDE_FILTER=` (keep your value). A leftover line with the
   old name **refuses the render** with a rename error (the entrypoint
   gate keeps the previous config running until you fix it — the LAN
   never goes down, but the new config won't apply).
2. **Add the include knob (recommended)** — `PRIORITY_INCLUDE_FILTER=日本|JP\d|^JP`
   (or your preferred region; see `.env.example`). Unset = the old
   all-minus-exclude behavior.
3. **Rewrite the DNS detour fragments** — every `#auto` in
   `DNS_FOREIGN_NAMESERVER` (and any custom entries) becomes `#All Nodes`,
   e.g. `https://1.1.1.1/dns-query#All Nodes,https://8.8.8.8/dns-query#All Nodes`.
   A leftover `#auto` fragment renders, but those resolvers dead-end at
   runtime (mihomo logs only) — foreign DNS would fail closed.
4. **Update COUNTRY_GROUPS** to the English names (or keep your own —
   interior spaces are now legal):
   `US Auto=美国|US\d|^US;Singapore Auto=新加坡|SG\d|^SG;Japan Auto=日本|JP\d|^JP;Taiwan Auto=台湾|TW\d|^TW;Philippines Auto=菲律宾|PH\d|^PH`
5. **Delete the `DNS_FALLBACK=` line** — the legacy profile it fed is
   gone. (Harmless if left: nothing reads it anymore.)
6. Confirm `DNS_CN_NAMESERVER` and `DNS_FOREIGN_NAMESERVER` are both set —
   the renderer now refuses to render without them.

## Expect once after upgrading

- **Dashboard pins reset once**: mihomo's `cache.db` keys selections by
  group NAME, so every renamed group falls back to its first member.
  Re-pin STREAMING (and any custom selections) in MetaCubeXD after the
  first start.
- The doctor's `dns_privacy` check still recognizes an old on-disk render
  (`v1`/`legacy`) until your first redeploy — that is the signal the new
  config has not applied yet.

## Deploy floor

The deployed mihomo image must be **≥ v1.19.27** (`empty-fallback` support;
since v1.19.28 it may name only a proxy — the template's `REJECT` stays
valid). Verify in the container during the deploy window:

    docker exec mihomo /mihomo -v

## 中文摘要

- 分组更名：`auto` → **All Nodes**，`auto-x` → **Priority Nodes**，国家组
  改为英文（US Auto……Philippines Auto），名称可含空格。
- 新增 `PRIORITY_INCLUDE_FILTER`（默认锁定日本）：默认线路不再在不同国家
  出口之间跳动；`AUTO_EXCLUDE_FILTER` **改名**为 `PRIORITY_EXCLUDE_FILTER`
  （残留旧键名会拒绝渲染——入口点守门会保持上一份配置运行）。
- STREAMING 选择器加入 Priority Nodes / All Nodes / 各国家组；Spotify、
  Tidal、Deezer、SoundCloud 规则并入 STREAMING。
- 清除全部迁移/兼容机制：分域解析 v2 成为唯一 DNS 配置档
  （`DNS_CN_NAMESERVER` + `DNS_FOREIGN_NAMESERVER` 必填，`DNS_FALLBACK`
  删除）；`#auto` 绕行片段需手工改为 `#All Nodes`。
- 升级后面板固定选择会**重置一次**（cache.db 按分组名存储），请在
  MetaCubeXD 中重新固定；部署窗口内确认容器内 mihomo 版本 ≥ v1.19.27。
