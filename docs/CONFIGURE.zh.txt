配置参考（.env）
================

网关从 ../syno-mihomo-gateway-data/.env 读取设置。代理订阅地址保存在
../syno-mihomo-gateway-data/config/subscription.txt 中。更新发布目录不会替换该同级目录。

通常你通过安装程序的提问来设置这些值。本文件说明每一项的含义，方便你查看或
手动编辑 .env。手动修改 .env 后，请用 "4) 修改现有部署" 应用更改，以便容器
在带健康检查与回滚的保护下重新部署。

该文件按 dotenv 数据解析，不会作为 shell 代码执行。每行只写一个 KEY=VALUE。
安装程序写入的引号值能安全保留密码和密钥中的空格、&、#、$、引号和反斜杠。

--------------------------------------------------------------------------
网络设置
--------------------------------------------------------------------------
  ROUTER_IP
      你的路由器 / 网关 IP 地址。部署时安装程序会根据所选接口自动填入此项，
      你只需确认。

  SUBNET_CIDR
      家庭局域网子网，CIDR 形式，例如 192.168.1.0/24。部署时根据所选接口
      自动填入，你只需确认。

  MIHOMO_IP
      给网关使用的一个空闲静态局域网 IP。它必须位于你的局域网子网内，且不能
      被其他设备占用。客户端把网关和 DNS 指向此 IP 即可被转发。部署时，若该
      IP 已在局域网上应答，安装程序会发出警告。

  PARENT_INTERFACE
      安装程序在接口扫描中保存的 macvlan 父网卡。留空 = 自动识别。

  TPROXY_DRIVER
      网关网络的二层驱动：macvlan（默认）或 ipvlan。请保持 macvlan——只有
      它能转发局域网客户端流量，而且即使父接口是 Open vSwitch，macvlan 子
      接口也能被局域网访问。ipvlan 只是安装程序在某些 OVS 环境下提供的仅限
      仪表盘的逃生口：它不会把局域网客户端流量经代理转发。

--------------------------------------------------------------------------
Mihomo TUN（透明网关）
--------------------------------------------------------------------------
  TUN_ENABLE
      透明网关 TUN。默认开启（true）。这正是透明网关：渲染配置带有使用
      "system" 栈的 tun: 块（并配 allow-lan: true 与 fake-ip DNS），局域网设备
      把网关和 DNS 指向 MIHOMO_IP 即可经机场出网。它们也可把 MIHOMO_IP:7890
      （http）/ :7891（socks）当作显式代理。"system" 栈不会截走控制器回包，因此
      MIHOMO_IP:CONTROLLER_PORT 上的仪表盘保持可访问——这才是 mihomo #1493 的
      真正修复方式，所以请保持 TUN 开启。设 TUN_ENABLE=false 会运行普通的非网关
      代理（仅可通过 redir/tproxy/mixed/socks 端口访问，不会透明拦截局域网客户端）。
      只接受小写 true/false。

  TUN_AUTO_REDIRECT
      仅当 TUN_ENABLE=true 时生效：可选的 Linux TCP 重定向优化。为兼容 DSM，
      默认 false。仅当安装程序的一次性 iptables 兼容性探测成功时才设为 true。
      只接受小写 true 和 false。

--------------------------------------------------------------------------
安装程序
--------------------------------------------------------------------------
  INSTALLER_LANG
      安装程序界面语言：en 或 zh。首次运行选择语言后保存，以后运行会跳过该
      提问。

--------------------------------------------------------------------------
面板 / 控制器
--------------------------------------------------------------------------
  WEB_UI_PORT
      发布在 NAS 上的面板端口。默认 8080。通过
      http://<NAS-IP>:<WEB_UI_PORT> 打开面板。

  CONTROLLER_PORT
      代理控制器端口。默认 9090。在面板中添加后端时使用。

  CONTROLLER_SECRET
      可选的面板鉴权密钥；留空 = 面板不认证。留空时安装程序会提议自动生成
      一个随机密钥（保存在此处）；部署完成的摘要会说明它的位置。再次运行
      时若已保存密钥，在提示处按回车保留现有密钥，输入 '-' 则清除。如已
      设置，请将其作为后端的密钥（Secret）填入面板。它不能包含 '|' 字符。

--------------------------------------------------------------------------
DNS 设置
--------------------------------------------------------------------------
  DNS_DEFAULT_NAMESERVER
  DNS_NAMESERVER
  DNS_FALLBACK
  DNS_CN_NAMESERVER
  DNS_FOREIGN_NAMESERVER
      以逗号分隔的 DNS 服务器列表。它们控制网关如何解析域名；错误的 DNS
      配置可能导致部署后网关处于不健康状态。出厂默认值对中国大陆网络
      安全、且做了隐私加固（分域解析 v2）：每个条目都是 IP 或基于 IP 的
      加密 DoH URL（不需要先解析任何域名），以 #auto 结尾的条目会改为
      经代理隧道（auto 节点组）拨号而不是直连。
      DNS_CN_NAMESERVER / DNS_FOREIGN_NAMESERVER 是分域解析对：中国列表
      内的域名只在国内解析，而其余全部——境外列表内的域名以及任何列表
      都没有收录的域名——只经隧道解析；机场彻底不可用时，这些查询会
      直接失败，绝不会泄漏给国内解析器。两项要么都设、要么都不设
      （都不设 = 旧的全国内行为；只设一个会拒绝渲染；从旧 .env 升级时，
      重新部署界面会主动提议帮你开启）。
      DNS_FALLBACK 只在旧配置档下使用——分域解析对设置后完全不再使用。
      DNS_NAMESERVER 必须保持国内、绝不能加 # 后缀：它还要在任何代理
      就绪之前解析机场节点域名，走隧道的条目会让全新启动卡死。

  谁能看到你的 DNS 查询（在出厂默认值下）
      * AliDNS（阿里）/ DNSPod（腾讯）：只有中国列表内的域名（经加密
        DoH），外加网关自身需要的少数引导主机。其余一概不会到达——
        在局域网设备上跑 dnsleaktest.com 扩展测试不会出现国内解析器。
      * 你的 ISP / 线路上的任何人：只能看到网关在与知名解析器 IP 做
        加密通信——看不到你查询的域名。
      * 机场运营方：它本来就代理的境外流量，现在包括你走隧道的 DNS。
      * Cloudflare / Google（境外解析器）：境外及未收录的域名，经隧道
        到达——它们看到的是机场的地址，不是你家的地址。
      doctor 会以 "dns_privacy" 报告当前生效的配置档。

  DNS_GEOIP_NO_RESOLVE
      设为 true 可让路由规则完全不再解析未命中的域名（连隧道也不走，
      零本地解析）。在 v2 默认值下这次解析本来就是私密的，因此保持
      false 即可——内置中国列表漏收的国内网站还能继续直连而不是走
      代理（见 TROUBLESHOOTING.zh.txt）。仅接受 true 或 false。

  SNIFFER_ENABLE
      true（默认）开启流量嗅探：在网关之外解析 DNS 的设备发来的裸 IP
      连接，会从连接本身读出真实域名，路由规则因此照常生效（流媒体
      回到自己的分组、被污染的解析在节点侧得到纠正）。这修复的是这类
      设备的"路由"；它们的 DNS 隐私仍需把设备 DNS 指向网关（见
      TROUBLESHOOTING.zh.txt"局域网客户端绕过网关 DNS"）。false 则
      不渲染嗅探块（v1.3.10 之前的行为）。

  AUTO_EXCLUDE_FILTER
      匹配该正则的机场节点会从 auto-x 分组中剔除；auto-x 是默认线路
      （面板 PROXY 选择器的第一项）。出厂默认剔除香港节点——很多网站
      封锁香港地址，而香港节点居多的套餐会让按延迟优选几乎总是选中
      香港。完整不过滤的 auto 分组仍在面板里，一键即可切回。若正则
      匹配了所有节点，auto-x 会直接拒绝流量而不是悄悄绕过代理直连
      （doctor 会报告）。留空则完全不渲染 auto-x 分组（你现有的配置
      逐字节不变）。不允许反引号。

  COUNTRY_GROUPS
      面板上一键"某国最快节点"：名称=正则;名称=正则 每项生成一个
      自动优选分组——面板显示"名称"，并始终选中名字匹配"正则"的
      最快节点。一个正则写多个国家即是多国分组。出厂示例适配以中文
      命名节点的机场（美国01、新加坡05……）——请按你的机场命名修改。
      匹配不到任何节点的分组被选中时会直接拒绝流量，而不是悄悄绕过
      代理（doctor 会报告）。名称不能含空格，也不能与 auto / auto-x / PROXY /
      STREAMING / DIRECT / REJECT 重名；反引号及格式错误的条目会
      拒绝渲染。留空则不生成任何分组（你现有的配置逐字节不变）。

      这些设置只作用于网关本身。NAS 自己的 DNS 在 DSM 控制面板 > 网络 >
      常规中设置，也必须指向你的网络可达的解析器——若 NAS 唯一的解析器
      不可达，它将无法访问厂商服务、也无法更新任何东西（见
      TROUBLESHOOTING.zh.txt；doctor 也会逐一探测 NAS 的解析器）。

通用
--------------------------------------------------------------------------
  TZ
      容器使用的时区，例如 Asia/Shanghai。

--------------------------------------------------------------------------
镜像来源
--------------------------------------------------------------------------
  REGISTRY_MODE
      镜像从何处拉取：
        acr    -（默认）从你自己的私有镜像源拉取。在主流公共仓库被封锁的
                 中国大陆推荐使用。
        docker - 上游公共仓库。仅在网络不受限的 NAS 上可用。

  MIHOMO_TAG / METACUBEXD_TAG / CF_TAG
      各组件的镜像标签。默认 "latest"。

  DOCKER_REGISTRY
      你的仓库地址（acr 模式下使用）。

  DOCKER_USERNAME
      你的仓库用户名（acr 模式下使用）。

  ACR_PASSWORD
      你的仓库密码（acr 模式下使用）。

  ACR_NAMESPACE
      你的仓库命名空间（acr 模式下使用）。

  在 acr 模式下，你只需提供一次仓库地址、命名空间、用户名和密码，安装程序会
  为你拼出完整的镜像引用。

--------------------------------------------------------------------------
自动更新
--------------------------------------------------------------------------
  UPDATE_ENABLED
      开启或关闭计划自动更新。

  UPDATE_IMAGES
      更新任务检查并拉取的镜像引用，以空格分隔。请保持它继承上面三个镜像
      变量（默认值 "${MIHOMO_IMAGE} ${METACUBEXD_IMAGE} ${CF_IMAGE}"）：
      一旦此列表不再原样包含 MIHOMO_IMAGE 和 METACUBEXD_IMAGE（以及已设置
      的 CF_IMAGE），更新器会拒绝运行。

  UPDATE_DENY_CONTAINERS
      可选的通配符排除列表（以空格分隔），作用于通用自动更新目标：名称
      匹配任一模式的已登记容器永远不会被自动更新。示例：
      "critical-* my-database"。详见 AUTO-UPDATE.zh.txt。

  UPDATE_SCHEDULE
      经过校验的五字段数字 cron 表达式。默认为每天 02:00。DSM 任务时间需
      与它一致。

  UPDATE_TZ
      更新器日志时间戳所用时区。DSM 按 NAS“区域选项”的系统时区触发任务。
      默认 Asia/Shanghai。

  DOCKER_READY_TIMEOUT / DOCKER_READY_INTERVAL
      计划或开机任务等待 Container Manager 的秒数和重试间隔。默认 120 / 5。

  EXPECTED_ARCH
      期望的 CPU 架构：amd64 或 arm64。设为与你的 NAS 一致，以便尽早发现
      架构不匹配。

  NOTIFY_WEBHOOK_URL
      可选的 webhook 地址（Bark / Gotify / 兼容 Slack 的 JSON POST），用于
      推送更新报告。设置后，失败和成功更新都会触发，与 DSM 自身的通知互相
      独立——它是可靠的通知渠道。留空则依赖 DSM 任务的“通过电子邮件发送
      运行详情”。

  NOTIFY_ON_NOCHANGE
      设为 1 时，即使本次更新运行没有任何变化也发送通知。默认 0（无变化
      时保持安静）。

  UPDATE_LOG / LOG_KEEP
      更新器的轮转日志文件及保留的轮转份数（默认 7）。UPDATE_LOG 为相对
      路径时会放到数据目录下：默认值 ./logs/auto-update.log 实际是
      ../syno-mihomo-gateway-data/logs/auto-update.log。

--------------------------------------------------------------------------
可选 / 进阶：隧道
--------------------------------------------------------------------------
  CF_* 设置
      隧道组件是可选且进阶的功能。除非确实需要隧道，否则请保持 CF_* 设置为
      默认值。部署时的镜像步骤会询问“是否同时管理 cloudflared 隧道容器？
      （进阶，可选）”——默认否。回答是则会收集 CF_TAG 和隧道令牌（按回车
      可复用正在运行容器的令牌）。

  CF_DNS
      可选：以逗号分隔的 IPv4 解析器，作为 docker --dns 传给隧道容器，
      使其不再继承 NAS 自身的 resolv.conf（NAS 的解析器不可达时，隧道
      会被悄悄拖死）。在下一次开通或蓝绿更新时生效
      （sudo sh scripts/gateway.sh update --yes）。留空保持现状。
      建议与 DNS_NAMESERVER 保持一致。

--------------------------------------------------------------------------
订阅地址
--------------------------------------------------------------------------
  代理订阅地址不在 .env 中，而是位于：
        ../syno-mihomo-gateway-data/config/subscription.txt
  安装程序会生成该文件，并在部署时询问该 URL。订阅地址错误或无法访问，是
  部署后网关不健康的常见原因。
