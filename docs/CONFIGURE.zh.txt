配置参考（.env）
================

网关从 ../syno-mihomo-gateway-data/.env 读取设置。代理订阅地址保存在
../syno-mihomo-gateway-data/config/subscription.txt 中。更新发布目录不会替换该同级目录。

通常你通过安装程序的提问来设置这些值。本文件说明每一项的含义，方便你查看或
手动编辑 .env。手动修改 .env 后，请用 "4) 修改现有部署" 应用更改，以便容器
在带健康检查与回滚的保护下重新部署。

该文件按 dotenv 数据解析，不会作为 shell 代码执行。每行只写一个 KEY=VALUE。
安装程序写入的引号值能安全保留密码和密钥中的空格、&、#、$、引号和反斜杠。

--------------------------------------------------------------------------
网络设置
--------------------------------------------------------------------------
  ROUTER_IP
      你的路由器 / 网关 IP 地址。部署时安装程序会根据所选接口自动填入此项，
      你只需确认。

  SUBNET_CIDR
      家庭局域网子网，CIDR 形式，例如 192.168.1.0/24。部署时根据所选接口
      自动填入，你只需确认。

  MIHOMO_IP
      给网关使用的一个空闲静态局域网 IP。它必须位于你的局域网子网内，且不能
      被其他设备占用。客户端把网关和 DNS 指向此 IP 即可被转发。部署时，若该
      IP 已在局域网上应答，安装程序会发出警告。

  PARENT_INTERFACE
      安装程序在接口扫描中保存的 macvlan 父网卡。留空 = 自动识别。

  TPROXY_DRIVER
      网关网络的二层驱动：macvlan（默认）或 ipvlan。请保持 macvlan——只有
      它能转发局域网客户端流量，而且即使父接口是 Open vSwitch，macvlan 子
      接口也能被局域网访问。ipvlan 只是安装程序在某些 OVS 环境下提供的仅限
      仪表盘的逃生口：它不会把局域网客户端流量经代理转发。

--------------------------------------------------------------------------
Mihomo TUN（透明网关）
--------------------------------------------------------------------------
  TUN_ENABLE
      透明网关 TUN。默认开启（true）。这正是透明网关：渲染配置带有使用
      "system" 栈的 tun: 块（并配 allow-lan: true 与 fake-ip DNS），局域网设备
      把网关和 DNS 指向 MIHOMO_IP 即可经机场出网。它们也可把 MIHOMO_IP:7890
      （http）/ :7891（socks）当作显式代理。"system" 栈不会截走控制器回包，因此
      MIHOMO_IP:CONTROLLER_PORT 上的仪表盘保持可访问——这才是 mihomo #1493 的
      真正修复方式，所以请保持 TUN 开启。设 TUN_ENABLE=false 会运行普通的非网关
      代理（仅可通过 redir/tproxy/mixed/socks 端口访问，不会透明拦截局域网客户端）。
      只接受小写 true/false。

  TUN_AUTO_REDIRECT
      仅当 TUN_ENABLE=true 时生效：可选的 Linux TCP 重定向优化。为兼容 DSM，
      默认 false。仅当安装程序的一次性 iptables 兼容性探测成功时才设为 true。
      只接受小写 true 和 false。

--------------------------------------------------------------------------
安装程序
--------------------------------------------------------------------------
  INSTALLER_LANG
      安装程序界面语言：en 或 zh。首次运行选择语言后保存，以后运行会跳过该
      提问。

--------------------------------------------------------------------------
面板 / 控制器
--------------------------------------------------------------------------
  WEB_UI_PORT
      发布在 NAS 上的面板端口。默认 8080。通过
      http://<NAS-IP>:<WEB_UI_PORT> 打开面板。

  CONTROLLER_PORT
      代理控制器端口。默认 9090。在面板中添加后端时使用。

  CONTROLLER_SECRET
      可选的面板鉴权密钥；留空 = 面板不认证。留空时安装程序会提议自动生成
      一个随机密钥（保存在此处）；部署完成的摘要会说明它的位置。再次运行
      时若已保存密钥，在提示处按回车保留现有密钥，输入 '-' 则清除。如已
      设置，请将其作为后端的密钥（Secret）填入面板。它不能包含 '|' 字符。

--------------------------------------------------------------------------
DNS 设置
--------------------------------------------------------------------------
  DNS_DEFAULT_NAMESERVER
  DNS_NAMESERVER
  DNS_CN_NAMESERVER
  DNS_FOREIGN_NAMESERVER
      以逗号分隔的 DNS 服务器列表。它们控制网关如何解析域名；错误的 DNS
      配置可能导致部署后网关处于不健康状态。出厂默认值对中国大陆网络
      安全、且做了隐私加固（分域解析 v2）：每个条目都是 IP 或基于 IP 的
      加密 DoH URL（不需要先解析任何域名），以 #All Nodes 结尾的条目
      会改为经代理隧道拨号而不是直连——走隐藏的 All Nodes 分组，一个
      只为充当此 DNS 锚点而保留的全池自动优选分组（面板不显示它；
      无论你在面板里怎么选，DNS 都照常工作）。既有安装的 DNS 值在
      精简分组模型下无需任何修改。
      DNS_CN_NAMESERVER / DNS_FOREIGN_NAMESERVER 是分域解析对：中国列表
      内的域名只在国内解析，而其余全部——境外列表内的域名以及任何列表
      都没有收录的域名——只经隧道解析；机场彻底不可用时，这些查询会
      直接失败，绝不会泄漏给国内解析器。两项都是必填——分域解析 v2 是
      唯一的 DNS 配置档，缺失任何一项渲染器都会拒绝渲染。
      DNS_NAMESERVER 必须保持国内、绝不能加 # 后缀：它还要在任何代理
      就绪之前解析机场节点域名，走隧道的条目会让全新启动卡死。

  谁能看到你的 DNS 查询（在出厂默认值下）
      * AliDNS（阿里）/ DNSPod（腾讯）：只有中国列表内的域名（经加密
        DoH），外加网关自身需要的少数引导主机。其余一概不会到达——
        在局域网设备上跑 dnsleaktest.com 扩展测试不会出现国内解析器。
      * 你的 ISP / 线路上的任何人：只能看到网关在与知名解析器 IP 做
        加密通信——看不到你查询的域名。
      * 机场运营方：它本来就代理的境外流量，现在包括你走隧道的 DNS。
      * Cloudflare / Google（境外解析器）：境外及未收录的域名，经隧道
        到达——它们看到的是机场的地址，不是你家的地址。
      doctor 会以 "dns_privacy" 报告当前生效的配置档。

  DNS_GEOIP_NO_RESOLVE
      设为 true 可让路由规则完全不再解析未命中的域名（连隧道也不走，
      零本地解析）。在 v2 默认值下这次解析本来就是私密的，因此保持
      false 即可——内置中国列表漏收的国内网站还能继续直连而不是走
      代理（见 TROUBLESHOOTING.zh.txt）。仅接受 true 或 false。

  SNIFFER_ENABLE
      true（默认）开启流量嗅探：在网关之外解析 DNS 的设备发来的裸 IP
      连接，会从连接本身读出真实域名，路由规则因此照常生效（流媒体
      回到自己的分组、被污染的解析在节点侧得到纠正）。这修复的是这类
      设备的"路由"；它们的 DNS 隐私仍需把设备 DNS 指向网关（见
      TROUBLESHOOTING.zh.txt"局域网客户端绕过网关 DNS"）。false 则
      不渲染嗅探块（v1.3.10 之前的行为）。

  FULL_PROXY_SOURCES
      可选的按设备全代理网段：以逗号分隔的 IPv4 地址/CIDR（裸 IP 视为
      /32；例如 192.168.1.240/28）。只有设置了才会渲染——不设置时渲染
      出的配置逐字节相同（该功能完全不可见）。设置后，面板上会出现一个
      "Full-Tunnel Devices" 选择器，且每个条目都会在 LAN 规则之后紧跟生成一条
      SRC-IP-CIDR 规则：网段内设备访问局域网目标仍然直连，而其余一切
      ——流媒体和国内网站一视同仁——都走 "Full-Tunnel Devices"（严格语义）。
      格式错误的值会拒绝渲染并点名此键——IPv6 条目、主机名、大于 255
      的段、大于 32 的前缀、重复项、空白、反引号（拒绝 IPv6 是有意
      为之：可路由的局域网 IPv6 会整个绕过这台仅 IPv4 的网关——见
      下文"全代理设备"一节的注意事项 3）。按设备切换模式在路由器侧
      完成（改一次 DHCP 固定 IP 保留），绝不需要重启网关。详见下文
      "全代理设备（FULL_PROXY_SOURCES）"一节。

  COUNTRY_GROUPS
      必填——面板的分组模型由它生成（留空会拒绝渲染；请从出厂默认
      值改起）：名称=正则;名称=正则 每项生成一个 "<Country> Auto"
      自动优选分组——面板显示"名称"，并始终选中名字匹配"正则"的
      最快节点——这些分组就是 "Exit Country" 选择器的全部成员，你在
      那里选择出口国家。第一项是开箱即用的默认国家（出厂默认把
      Japan Auto 放在第一位）。一个正则写多个国家即是多国分组。出厂
      示例适配以中文命名节点的机场（美国01、新加坡05……）——请按你
      的机场命名修改。匹配不到任何节点的分组被选中时会直接拒绝流量，
      而不是悄悄绕过代理（doctor 会标记；若空掉的正是 Exit Country
      当前所选的国家，默认线路流量会被拦截——修正则期间可先在那里
      改选另一个国家）。名称可含中间空格（"Japan Auto"），但不能以
      空格开头/结尾，也不能与内建、保留或已退役的分组名重名
      （All Nodes / Exit Country / Routing Mode / Streaming Unlock /
      DIRECT / REJECT / ……——渲染错误会点名冲突项；完整清单见
      .env.example）；反引号及格式错误的条目会拒绝渲染。

  面板上的代理分组
      Routing Mode 是路由规则指向的模式选择器：默认 Exit Country，
      另有 DIRECT / REJECT / 机场原始节点（当某网站连同国 IP 变化
      也要计较时，在这里固定一个具体节点即可）。Streaming Unlock 是
      针对解锁敏感站点的按服务选择器（Netflix + 各音频流媒体；默认
      Routing Mode）。Exit Country 选择出口国家：成员正是上面的
      "<Country> Auto" 分组，因此常规流量保持单一出口 IP，出口国家
      绝不会自行跳变。All Nodes 仍然存在但已隐藏——它只为上面的
      DNS 绕行充当锚点。命名图例："<X> Auto" = 自动优选
      （该国家内最快节点）；选择器采用功能命名（#58）：
      Routing Mode = 主模式开关，Streaming Unlock = 流媒体出口固定，
      Exit Country = 代理出口国家，Full-Tunnel Devices = 按设备
      强制代理段。

  已移除的键
      旧的优先节点圈定/剔除过滤对（及更早的 AUTO_EXCLUDE_FILTER）
      已经移除——由 "<Country> Auto" 分组加 Exit Country 取代。
      .env 中仍残留任何这类行都会拒绝渲染；错误会点名需要删除的行
      （期间网关继续运行上一份有效配置）。本次升级后，面板选择会
      重置一次（保存的选择按分组名记录）——请在面板里重新固定。

      这些设置只作用于网关本身。NAS 自己的 DNS 在 DSM 控制面板 > 网络 >
      常规中设置，也必须指向你的网络可达的解析器——若 NAS 唯一的解析器
      不可达，它将无法访问厂商服务、也无法更新任何东西（见
      TROUBLESHOOTING.zh.txt；doctor 也会逐一探测 NAS 的解析器）。

镜像来源
--------------------------------------------------------------------------
  REGISTRY_MODE
      镜像从何处拉取：
        acr    -（默认）从你自己的私有镜像源拉取。在主流公共仓库被封锁的
                 中国大陆推荐使用；也是群晖与 install-pi.sh 的默认值。
        docker - 上游公共仓库（多架构）。需要不受限的网络；在通用 Linux
                 主机上它是 install-linux.sh 提供的默认选项。

  MIHOMO_TAG / METACUBEXD_TAG / CF_TAG
      各组件的镜像标签。默认 "latest"。

  DOCKER_REGISTRY
      你的仓库地址（acr 模式下使用）。

  DOCKER_USERNAME
      你的仓库用户名（acr 模式下使用）。

  ACR_PASSWORD
      你的仓库密码（acr 模式下使用）。

  ACR_NAMESPACE
      你的仓库命名空间（acr 模式下使用）。

  在 acr 模式下，你只需提供一次仓库地址、命名空间、用户名和密码，安装程序会
  为你拼出完整的镜像引用。

--------------------------------------------------------------------------
自动更新
--------------------------------------------------------------------------
  UPDATE_ENABLED
      开启或关闭计划自动更新。

  UPDATE_IMAGES
      更新任务检查并拉取的镜像引用，以空格分隔。请保持它继承上面三个镜像
      变量（默认值 "${MIHOMO_IMAGE} ${METACUBEXD_IMAGE} ${CF_IMAGE}"）：
      一旦此列表不再原样包含 MIHOMO_IMAGE 和 METACUBEXD_IMAGE（以及已设置
      的 CF_IMAGE），更新器会拒绝运行。

  UPDATE_DENY_CONTAINERS
      可选的通配符排除列表（以空格分隔），作用于通用自动更新目标：名称
      匹配任一模式的已登记容器永远不会被自动更新。示例：
      "critical-* my-database"。详见 AUTO-UPDATE.zh.txt。

  UPDATE_SCHEDULE
      经过校验的五字段数字 cron 表达式。默认为每天 02:00。DSM 任务时间需
      与它一致。

  UPDATE_TZ
      更新器日志时间戳所用时区。DSM 按 NAS“区域选项”的系统时区触发任务。
      默认 Asia/Shanghai。

  DOCKER_READY_TIMEOUT / DOCKER_READY_INTERVAL
      计划或开机任务等待 Container Manager 的秒数和重试间隔。默认 120 / 5。

  EXPECTED_ARCH
      期望的 CPU 架构：amd64、arm64 或 arm（32 位，例如 armv7 树莓派）。
      设为与你的机器一致，以便尽早发现架构不匹配；install-linux.sh 与
      install-pi.sh 会自动把它固定为主机架构。

  NOTIFY_WEBHOOK_URL
      可选的 webhook 地址（Bark / Gotify / 兼容 Slack 的 JSON POST），用于
      推送更新报告。设置后，失败和成功更新都会触发，与 DSM 自身的通知互相
      独立——它是可靠的通知渠道。留空则依赖 DSM 任务的“通过电子邮件发送
      运行详情”。

  NOTIFY_ON_NOCHANGE
      设为 1 时，即使本次更新运行没有任何变化也发送通知。默认 0（无变化
      时保持安静）。

  UPDATE_LOG / LOG_KEEP
      更新器的轮转日志文件及保留的轮转份数（默认 7）。UPDATE_LOG 为相对
      路径时会放到数据目录下：默认值 ./logs/auto-update.log 实际是
      ../syno-mihomo-gateway-data/logs/auto-update.log。

--------------------------------------------------------------------------
Lite 模式（通用 Linux / 树莓派）
--------------------------------------------------------------------------
  由通用 Linux / 树莓派安装器（install-linux.sh / install-pi.sh）与 lite
  更新器使用。在群晖 NAS 上无害且不生效（.env.example 中以注释形式
  提供）。

  GH_MIRROR
      施加在上游发布下载地址上的镜像前缀（mihomo 程序、面板、最新版本
      号解析），形如 <镜像站>/<完整上游地址>。留空 = 直连。

  MIHOMO_VERSION
      固定 lite 安装/更新使用的 mihomo 发布版本号。留空表示"最新"。

  MIHOMO_SHA256
      固定版本时可选的完整性锚点（上游不发布校验和）：设置后，下载的
      压缩包必须匹配才会安装。下载经过第三方镜像时强烈建议设置。

  EXTERNAL_UI_DIR
      mihomo 托管面板的目录。仅在设置时渲染进配置：lite 向导会设置它，
      群晖/compose 路径保持不设，其渲染结果因此逐字节不变。

--------------------------------------------------------------------------
可选 / 进阶：隧道
--------------------------------------------------------------------------
  CF_* 设置
      隧道组件是可选且进阶的功能。除非确实需要隧道，否则请保持 CF_* 设置为
      默认值。部署时的镜像步骤会询问“是否同时管理 cloudflared 隧道容器？
      （进阶，可选）”——默认否。回答是则会收集 CF_TAG 和隧道令牌（按回车
      可复用正在运行容器的令牌）。

  CF_DNS
      可选：以逗号分隔的 IPv4 解析器，作为 docker --dns 传给隧道容器，
      使其不再继承 NAS 自身的 resolv.conf（NAS 的解析器不可达时，隧道
      会被悄悄拖死）。在下一次开通或蓝绿更新时生效
      （sudo sh scripts/gateway.sh update --yes）。留空保持现状。
      建议与 DNS_NAMESERVER 保持一致。

--------------------------------------------------------------------------
全代理设备（FULL_PROXY_SOURCES）
--------------------------------------------------------------------------
  可选。设置 FULL_PROXY_SOURCES 会保留一小段 IPv4 网段，段内设备跳过
  上面的智能分流：每个条目会在 LAN 规则之后紧跟拼入一条 SRC-IP-CIDR
  规则，因此段内设备访问局域网目标仍然直连，而其余一切——流媒体和
  国内网站一视同仁——都走 "Full-Tunnel Devices" 分组（严格语义：这些来源没有
  国内直连捷径）。不设置则什么都不渲染，配置逐字节不变——在你主动
  启用之前该功能完全不可见。

  "Full-Tunnel Devices" 分组是一个面板选择器。成员：
      * Routing Mode（默认）——网段跟随面板上 Routing Mode 的当前选择。
      * 每个 "<Country> Auto" 分组——独立于主 "Exit Country" 的
        选择，单独固定网段的出口国家。
      * REJECT——断网开关：一键让整个网段下线。
  有意不设 DIRECT 成员：面板上的一次点击永远不可能让段内设备悄悄
  脱离代理——离开网段只能在路由器侧完成（见下），绝不是选择器的
  某个状态。

  前提：DHCP 固定 IP 保留是路由器的职责。网关不管理租约——它只匹配
  来源 IP。请在路由器上保留一小段固定 IP（例如 192.168.1.240/28），
  并把 FULL_PROXY_SOURCES 设成它。

  设备进出网段的切换在路由器侧完成——已在 UniFi（按客户端的
  Fixed IP 保留）上验证，任何支持按客户端 DHCP 保留的路由器做法
  相同：
      1. 把设备的固定 IP 保留改到网段内，并让设备重连（开关 Wi-Fi /
         重新插网线）以获取新租约：该设备即进入全代理。
      2. 把保留改回网段外并重连：恢复正常的规则分流。
  全程不涉及网关或容器重启——模式切换完全是路由器侧的一次租约
  变更。

  修改网段本身（即 .env 里的值）就是一次普通的 .env 编辑，用
  "4) 修改现有部署" 应用更改
  （sudo sh scripts/gateway.sh redeploy --yes）。渲染门禁照常生效：
  格式错误的值会拒绝渲染并点名 FULL_PROXY_SOURCES，网关继续运行上
  一份有效配置。以下情况都会大声拒绝：IPv6 条目、主机名、大于 255
  的段、大于 32 的前缀、重复项、空白、反引号。拒绝 IPv6 是有意为之
  ——匹配仅限 IPv4，而可路由的局域网 IPv6 会整个绕过网关（见下方
  注意事项 3）。

  doctor 新增 "full_proxy" 检查：disabled（未使用该键）/ ok /
  parity-drift（渲染出的规则与该键不再一致——你的网段改动尚未生效，
  请重新部署）/ chain-violation（网段来源的一条非局域网流量绕过了
  Full-Tunnel Devices）。

  注意事项：
      1. 段内设备访问国内网站/应用会从境外出口。在 fake-ip 下，段内
         设备的域名由节点在远端解析——国内域名"正确地"走了隧道；
         国内服务会看到境外出口并相应降级。这正是严格语义的设计
         行为，不是泄漏。
      2. QUIC/UDP 落空。出口节点不支持 UDP 转发时，UDP 流量会在规则
         引擎处穿过 SRC-IP-CIDR 规则落到后面——命中国内列表的目标
         随即直连；浏览器会改用走代理的 TCP 重试。doctor 的
         full_proxy 检查会标记这类流量；只有当某家机场的 UDP 确实
         不可靠时，才会提供可选的 UDP 拦截开关。
      3. 该保证的前提是局域网没有可路由的 IPv6（doctor 的
         ipv6_bypass 检查）——双栈设备的全局 IPv6 会整个绕过这台仅
         IPv4 的网关。

--------------------------------------------------------------------------
订阅地址
--------------------------------------------------------------------------
  代理订阅地址不在 .env 中，而是位于：
        ../syno-mihomo-gateway-data/config/subscription.txt
  安装程序会生成该文件，并在部署时询问该 URL。订阅地址错误或无法访问，是
  部署后网关不健康的常见原因。
