自动更新
========

网关可以按计划自动更新镜像。更新脚本会拉取最新镜像、仅重新部署发生变化的
部分、运行健康检查，并在新镜像不健康时自动回滚。

你可以从安装程序进行设置：

      sudo sh ./install.sh
      -> 2) 设置自动更新（cron）

--------------------------------------------------------------------------
推荐：DSM 任务计划（可在系统升级后保留）
--------------------------------------------------------------------------
在群晖上，推荐且能在系统升级后保留的更新计划方式，是在 DSM 中使用用户自定义
脚本：

      控制面板 -> 任务计划 -> 新增 -> 计划的任务
              -> 用户定义的脚本

以 root 身份运行。当你选择选项 2 时，安装程序会打印需要填入的准确设置
（命令行、计划时间和用户）。把它们复制到任务计划的任务中。

为什么更推荐 GUI 任务：crontab 行可能在系统升级时被 DSM 覆盖；而任务计划中的
任务可在升级后保留。

--------------------------------------------------------------------------
"设置自动更新" 提供的选项
--------------------------------------------------------------------------
  * 打印准确的 DSM 任务计划设置（推荐方式）。
  * 运行 DRY-RUN（试运行）：校验整条更新流程但不做任何改动。在信任计划任务
    之前，用它确认一切已正确配置。
  * 可选地安装一条 FALLBACK（回退）crontab 行。它能用，但 DSM 可能在系统
    升级时将其覆盖，因此更推荐 GUI 任务。

--------------------------------------------------------------------------
相关 .env 设置
--------------------------------------------------------------------------
  UPDATE_ENABLED    - 开启/关闭计划更新。
  UPDATE_SCHEDULE   - 更新运行时机的 cron 表达式。
  UPDATE_TZ         - 计划使用的时区。
  EXPECTED_ARCH     - amd64 或 arm64；设为与你的 NAS 一致。

完整的设置参考见 CONFIGURE.zh.txt。

--------------------------------------------------------------------------
更新脚本每次运行时所做的事
--------------------------------------------------------------------------
  1. 从你配置的镜像来源拉取最新镜像。
  2. 与正在运行的版本比较，仅重新部署发生变化的组件。
  3. 对新容器运行健康检查。
  4. 若新镜像不健康，回滚到上一个可用镜像。

--------------------------------------------------------------------------
更新脚本退出码
--------------------------------------------------------------------------
  0  成功或无操作（无需更新）。
  2  部分失败（部分组件已更新，其余未更新）。
  3  配置 / 预检错误（检查 .env 与先决条件）。
  4  已有一次运行正在进行中。
  5  仓库登录失败（检查你的仓库凭据）。

如果计划任务报告非零退出码，请参见 TROUBLESHOOTING.zh.txt，并查看 logs/
文件夹下的自动更新日志。

--------------------------------------------------------------------------
日志
--------------------------------------------------------------------------
更新脚本会在 logs/ 文件夹下写入一份自动更新日志。计划任务运行后请查看它，
确认有哪些变化以及是否发生了回滚。
