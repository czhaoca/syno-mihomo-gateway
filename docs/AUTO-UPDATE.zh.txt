自动更新
========

网关可以按计划自动更新镜像。更新脚本会拉取最新镜像、仅重新部署发生变化的
部分、运行健康检查，并在新镜像不健康时自动回滚。

你可以从安装程序进行设置：

      sudo sh ./install.sh
      -> 3) 设置自动更新（cron）

--------------------------------------------------------------------------
选择时间与时区
--------------------------------------------------------------------------
cron 设置会以 HH:MM 形式询问每天的运行时间。默认计划为每天 02:00。随后它会
让你从菜单中选择时区（默认 Asia/Shanghai，另有其他常见时区，或自行键入）。

--------------------------------------------------------------------------
推荐：DSM 任务计划（可在系统升级后保留）
--------------------------------------------------------------------------
在群晖上，推荐且能在系统升级后保留的更新计划方式，是在 DSM 中使用用户自定义
脚本：

      控制面板 -> 任务计划 -> 新增 -> 计划的任务
              -> 用户定义的脚本

以 root 身份运行。当你选择 DSM 网页界面方式时，安装程序会打印需要填入的准确
设置（命令行、计划时间和用户）。把它们复制到任务计划的任务中。

为什么更推荐 GUI 任务：crontab 行可能在系统升级时被 DSM 覆盖；而任务计划中的
任务可在升级后保留。

--------------------------------------------------------------------------
如何安排计划（一个明确的选择）
--------------------------------------------------------------------------
在你选定时间与时区之后，安装程序会让你选择如何安排计划：

  1) 通过 DSM 网页界面（任务计划）设置 - 推荐，且能在系统升级后保留。安装
     程序会打印需要填入的准确 "控制面板 -> 任务计划" 设置。
  2) 现在通过 CLI 安装一条 crontab 条目 - 快捷，但可能在 DSM 系统升级时被
     覆盖。
  3) 仅运行 DRY-RUN（试运行）进行校验 - 校验整条更新流程但不做任何改动。在
     信任计划任务之前，用它确认一切已正确配置。

--------------------------------------------------------------------------
重要：时区与实际触发时间
--------------------------------------------------------------------------
你选择的时区用于标注任务内日志的时间戳。对于 CRONTAB 条目，实际触发时间遵循
NAS 系统时区，而非你选择的时区；DSM 任务计划的任务则按 DSM 自身的区域选项
时间触发。无论哪种方式，请正确设置 NAS 时间，使任务在你预期的时刻运行。

--------------------------------------------------------------------------
相关 .env 设置
--------------------------------------------------------------------------
  UPDATE_ENABLED    - 开启/关闭计划更新。
  UPDATE_SCHEDULE   - 更新运行时机的 cron 表达式（默认每天 02:00）。
  UPDATE_TZ         - 计划使用的时区（默认 Asia/Shanghai）。
  EXPECTED_ARCH     - amd64 或 arm64；设为与你的 NAS 一致。

完整的设置参考见 CONFIGURE.zh.txt。

--------------------------------------------------------------------------
更新脚本每次运行时所做的事
--------------------------------------------------------------------------
  1. 从你配置的镜像来源拉取最新镜像。
  2. 与正在运行的版本比较，仅重新部署发生变化的组件。
  3. 对新容器运行健康检查。
  4. 若新镜像不健康，回滚到上一个可用镜像。

--------------------------------------------------------------------------
更新脚本退出码
--------------------------------------------------------------------------
  0  成功或无操作（无需更新）。
  2  部分失败（部分组件已更新，其余未更新）。
  3  配置 / 预检错误（检查 .env 与先决条件）。
  4  已有一次运行正在进行中。
  5  仓库登录失败（检查你的仓库凭据）。

如果计划任务报告非零退出码，请参见 TROUBLESHOOTING.zh.txt，并查看 logs/
文件夹下的自动更新日志。

--------------------------------------------------------------------------
日志
--------------------------------------------------------------------------
更新脚本会在 logs/ 文件夹下写入一份自动更新日志。计划任务运行后请查看它，
确认有哪些变化以及是否发生了回滚。
