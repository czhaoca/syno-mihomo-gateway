Installing the Mihomo Gateway
=============================

This guide walks you through installing the gateway on a Synology NAS from
this release bundle.

Overview of the steps
---------------------
  1. Transfer this folder to the NAS.
  2. Place it under the Docker shared folder.
  3. Run install.sh, pick a language, and choose "Deploy".
  4. Answer the deploy prompts.
  5. Verify from another device on the LAN.

--------------------------------------------------------------------------
1. Transfer this folder to the NAS
--------------------------------------------------------------------------
Copy this unpacked folder onto the NAS using File Station, an SMB share, or
scp. You can put it anywhere at first; the installer will tell you if it needs
to move.

--------------------------------------------------------------------------
2. Place it under the Docker shared folder (required)
--------------------------------------------------------------------------
The folder MUST live under the Docker shared folder (typically
/volume1/docker; your volume number may differ). This is because the Docker
daemon bind-mounts files from wherever docker-compose.yml lives.

The installer checks this FIRST. If the folder is in the wrong place, it
prints exact instructions, for example:

      sudo mv "<here>" "/volume1/docker/"

Then re-run install.sh from the new location. You can also move it in File
Station by dragging it into the "docker" shared folder.

--------------------------------------------------------------------------
3. Run the installer
--------------------------------------------------------------------------
Open a terminal in the folder and run:

      sudo sh ./install.sh

Root is required because deployment creates /dev/net/tun, creates the macvlan
network, and runs docker. If you forget sudo, the installer asks you to
re-run with:

      sudo sh ./install.sh

The FIRST time you run it, the installer asks for a language before anything
else:

      1) English   2) Chinese

The whole installer then runs in that language. The choice is saved (as
INSTALLER_LANG in .env), so later runs skip this prompt and go straight to
the menu. You can press Ctrl-D at any prompt or menu to quit cleanly.

You will see the menu:

      1) Deploy the gateway (first run, end-to-end)
      2) Redeploy (reuse saved settings; fix a conflicting IP)
      3) Set up automatic updates (cron)
      4) Modify an existing deployment
      5) Quit

Choose 1 for a first install.

--------------------------------------------------------------------------
4. The deploy flow (option 1)
--------------------------------------------------------------------------
Deploy walks through these steps in order:

  1.  Prepare config files - seeds .env and config/subscription.txt.
  2.  Scan the network interface FIRST - you pick the LAN interface from the
      scanned list (or type one). The installer then AUTO-FILLS the router /
      gateway IP and the LAN subnet (CIDR) from that interface, so you just
      confirm them rather than typing them.
  3.  Gateway IP & DNS - set mihomo's LAN IP and the DNS servers. When you set
      mihomo's IP, the installer checks whether that IP is already answering
      on the LAN (e.g. a DHCP-assigned device) and warns you before
      deploying. You can pick another IP or override the warning.
  4.  Image source - choose where images are pulled from (see below).
  5.  Subscription URL - the proxy subscription that feeds your rules.
  6.  Checks - validates prerequisites.
  7.  Existing-resource choices - independently preserve, automatically
      dismantle verified project containers/macvlan, or handle them manually.
  8.  Log in, pull/check images, validate the rendered config and Compose.
  9.  Apply the chosen cleanup only after every non-destructive check passes.
  10. Create or reuse the matching network.
  11. Start containers and run the health check.
  12. Summary - prints the dashboard URL and the values you need.

Automatic cleanup never removes an ambiguous container, a network with an
unrelated attachment, or the external cloudflared container. Manual mode only
prints commands and lets you rescan; it performs no cleanup itself.

Image source (asked during deploy)
  * acr (DEFAULT, recommended in mainland China): pulls from YOUR private
    image mirror. Enter the registry host, namespace, username, and password
    ONCE; the installer builds the full image references for you.
  * docker (upstream public registries): only works on a NAS with unfiltered
    internet. The installer warns and asks you to confirm. Behind a blocking
    firewall this mode will fail.

Interface selection (scanned first)
  The installer scans the NAS network interfaces, shows each with its IPv4,
  marks the auto-detected LAN interface as recommended, and lets you pick a
  number or type an interface name manually. The interface you pick drives the
  auto-filled router IP and subnet, and is saved as PARENT_INTERFACE in .env.

IP-conflict check (when you set mihomo's IP)
  The installer probes the LAN to see if the IP you chose for mihomo is
  already in use. If something answers, it warns you so a silent conflict does
  not break the deploy. Choose a free IP, or override to keep the one you
  entered. If a deploy still fails because the IP was taken, use Redeploy
  (menu item 2, below) to re-apply with a different IP without re-entering
  everything.

--------------------------------------------------------------------------
4b. Redeploy (menu option 2)
--------------------------------------------------------------------------
Redeploy reuses everything already saved in .env - router, DNS, ports,
subscription, and image settings - so you do NOT re-enter them. Use it when a
deploy failed because the chosen IP was taken, or when you just want to
re-apply. It offers:

  * Redeploy as-is.
  * Change mihomo's LAN IP, then redeploy.
  * Re-pick the network interface, then redeploy.

This is the recommended fix for an IP conflict.

--------------------------------------------------------------------------
5. Verify from ANOTHER device (important)
--------------------------------------------------------------------------
Because the gateway uses macvlan, the NAS host cannot reach the gateway's own
LAN IP. Do these checks from a DIFFERENT LAN device:

  1. Open the dashboard:   http://<NAS-IP>:<WEB_UI_PORT>   (default 8080).
     (This URL is also reachable from the NAS itself.)
  2. In the dashboard, add a backend:
        Host   = <MIHOMO_IP>
        Port   = <CONTROLLER_PORT>     (default 9090)
        Secret = <CONTROLLER_SECRET>   (if you set one)
  3. To route a client, set that client's gateway and DNS to <MIHOMO_IP>,
     then browse the internet from it to confirm traffic flows.

Run the structural diagnostic at any time with:

      sudo sh scripts/doctor.sh --egress

--------------------------------------------------------------------------
After install
--------------------------------------------------------------------------
  * Review CONFIGURE.txt for the full .env reference.
  * Set up automatic updates with AUTO-UPDATE.txt.
  * If anything fails, see TROUBLESHOOTING.txt.
  * An install log is written under the logs/ folder.
